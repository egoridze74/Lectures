## Историческая справка
В 1970-ом, когда появилась [[Характеристики UNIX-подобных систем|UNIX]]-система, интерпретатор назывался `sh`. 
- Далее, почти сразу после выхода, появилась новая версия - B-shell (Stephen Bourne, Bell Laboratories). Это программа известна под именем `bash`
- Потом появился C-shell (Bill Joy, Калифорнийский университет)
- Далее возник K-shell (David Korn, Bell Labs). Объединяет возможности B-shell и C-shell
- В 2001 захотелось придумать стандартное решение для shell. В результате появился P-shell (Portable Operating System Interface - POSIX)
- Чёт потом появился F-shell (Paul Falstadt Co Apple)

## Функции интерпретатора shell
1. **Обработка имён файлов, определённых через метасимволы**:
	$*$ - произвольная цепочка символов (например, можно вывести содержимое файлов p1, p2, p3, p4 при помощи `pr *` или `pr p*`)
	? - один произвольный символ (например, можно вывести содержимое этих же файлов при помощи `pr p?`)
	[] - альтернативный символ подстановки (например, можно вывести содержимое этих же файлов при помощи `pr p [ 1 2 3 4 ]` или `pr p [ 1-4 ]`)
2. **Интерпретация команд**
3. **Переадресация ввода-вывода**
	- > - знак **жёсткой переадресации** (например, вывести содержимое папки в файл a.txt - это `ls > a.txt`). Содержимое полностью переписывается
	- >> - знак **мягкой переадресации** (например, вывести содержимое  папки в файл a.txt - это `ls >> a.txt`). Содержимое дописывается в конец
	- < - знак **жёсткой переадресации**. Иногда его упускают, но это зависит от специфики команды
	- << - знак **мягкой переадресации**. Но это очень редкая ситуация. Обычно мягкая переадресация не используется
	- | - знак переадресации вывода между программами - конвейера (например, `ls | wc -l` - выводит количество строк при выводе ls, а `ls -a | wc -w` - количество файлов в директории)
	- Ещё для объединения команд юзают значки "и" и "или"
4. Создание среды пользователя
5. Поддержка командного языка программирования
	Пример создания переменных на shell:
```
a = 2
b = sasha
echo $b // напишет значение переменной b
```

Ещё переменные можно создавать так:
```
read a b c
egor 5 // печатаем вот это на клаве и жмём Enter
echo $a // выдаст "egor"
echo $b // выдаст "5"
echo $с // выдаст пустую строку

read a
egor 5 2025 // печатаем вот это на клаве и жмём Enter
echo $a // выдаст "egor 5 2025"

cmp a.c b.c
echo $? // выдаст 0, 1 или 2
// 0 - совпадают
// 1 - разные
// 2 - одного из файлов нет или недостаточно прав доступа
```

## Ключи разных интерпретаторов shell
- `$$` - идентификатор текущего процесса
- `$?` - возвращаемое значение последней выполненной программы
- `$#` - количество внешних аргументов вызываемой программы
- `#` - комментарий
- `$<номер>` - значение внешнего аргумента под номером <номер>

## Обозначения в shell
Интерпретатор shell допускает следующие обозначения:
- $`\dots`$ - подстановка результата выполнения команды (пример: $a = `ls`$, `echo $a`). Можно ещё `a = $(ls -a)`, но это будет работать не во всех интерпретаторах shell
- `(...)` - формирование процесса (дополнительная нагрузка на процесс)
- `\` - отменяет специальный смысл символа, следующего за ним
- `;` - позволяет в одной строке реализовывать несколько команд
- `"..."` - отменяют специальный смысл всех символов внутри себя, за исключением: `\, ", $, '`
- `'...'` - отменяют специальный смысл всех символов внутри себя без исключений
