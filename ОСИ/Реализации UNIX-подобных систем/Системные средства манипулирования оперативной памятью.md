### Системные вызовы
- `int brk(void *end_data_segment)`
- `int sbrk()`
- `void mmap(void *start, size_t length, int prot, int flags, int fd, off_t offset)`
	start - адрес, с которого будет начинаться файл
	length - количество байтов, начиная с offset, которое мы хотим отобразить
	offset - указатель 
- `void unmap()`
- `int shmget(key_t key, int size, int flag)`
	позволяет создать или открыть разделяемую область памяти ядра размером `size`
- `shmat()` - присоединить память к адресному пространству, `shamdt()` - отсоединить
- `shmctl()` - работа с памятью (поменять владельца, права доступа)

### Утилиты
- `free` - предоставляет данные о перераспределении оперативной памяти
	total - объем оперативки в байтах
	used - объем занятой оперативки в байтах
	free - сколько свободно
	shared - сколько использовано под разделяемую память
	buff/cache - сколько под кэши
	swap - сколько под своппинг
- `atop` - инфа по процессу (где находится, сколько памяти занимает)
- `memstat` - состояние памяти (сколько памяти занимает каждый процесс)
- `htop` - Надстройка над `atop` Позволяет посмотреть объем резидентной (невыгружаемой) памяти, размер библиотек
- `vmstat` - можно посмотреть, какие процессы находятся в оперативке, а какие во внешней, количество прерываний в секунду, время простоя, время работы страничного демона, процессорное время для каждого пользака
