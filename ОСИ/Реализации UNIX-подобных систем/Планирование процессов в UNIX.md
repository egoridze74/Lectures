#оси 
**Планирование процессов** - это определение последовательности выполнения [[Процесс. Состояния процесса|процессов]]. Оно предусматривает 2 алгоритма:
1) Низкоуровневый алгоритм - выбирает следующий процесс из набора процессов в памяти, **готовых к работе**, то есть, отвечает за выполнение процессов
2) Высокоуровневый алгоритм - перемещает процессы из [[Оперативная память|оперативной памяти]] во внешнюю и обратно, то есть осуществляет "swapping"

Все эти алгоритмы реализованы в процессе с pid = 0 ([[Создание процессов в UNIX|swapper]]).

## Низкоуровневый алгоритм
Низкоуровневый алгоритм использует несколько очередей, чтобы быстрее находить процессы с наименьшим приоритетом.

Когда запускается низкоуровневый алгоритм, он: 
1) Начинает искать очереди с самым высоким приоритетом, пока не найдёт очередь, где есть $>=1$ процесс
2) Запускает самый первый процесс в очереди
3) Если процесс использует весь свой квант времени, то он помещается в конец своей очереди
4) Алгоритм планирования запускается снова

Раз в секунду приоритет каждого процесса пересчитывается по формуле:
$$PRI = \mu CPU + \nu NICE + BASE$$, где 
- CPU - среднее значение тиков таймера, в течение которых процесс выполнялся. При каждом тике таймера счётчик использования процессорного времени в дескрипторе процесса увеличивается на 1
- NICE - пользовательская составляющая приоритета процесса (принимает значения $[-20; 20]$, по дефолту 0)
- BASE - отрицательное число, которое добавляется к приоритету процесса при его выходе из состояния блокированного. Оно жёстко прописано в компьютере и неизменяемо

После пересчёта состав очередей процессов на выполнение меняется.
