## Классификация утилит
- Внешние утилиты требуют выполнения 2 или более процессов (`ls, grep, cp`)
- Внутренние утилиты выполняются в рамках одного процесса

## Внутренние утилиты
- `:` - "ноль функция" или "ноль программа", то есть утилита, которая всегда возвращает 0. Каждый из интерпретаторов имеет эту утилиту.
- `break` - выполняется в конструкциях цикла для выхода из цикла
- `cd` - позволяет менять текущий каталог ("change directory")
- `continue` - продолжить цикл со следующей итерации
- `echo` - программа, которая выводит в стандартный файл вывода список своих внешних аргументов
- `eval` - выполняет результирующую команду с разными аргументами
- `exec` - выполнить программу, но не в рамках этой shell среды
- `exit` - выход из интерпретатора shell
- `export` - позволяет экспортировать shell переменные между скриптами
- `pwd` - отображает имя текущего каталога
- `read` - считывает цепочку символов из стандартного ввода
- `readonly` - преобразовывает указанную переменную в переменную только для чтения (константу)
- `return` - выход из функции с отображением кода возврата
- `shift` - смещает влево командную строку аргументов
- `test` - оценивает условное выражение
- `trap` - позволяет перехватить у ОС обработку прерываний
- `ulimit` - отображает (устанавливает) ресурсы данной среды shell 
- `umask` - отображает (устанавливает) режимы создания файлов
- `unset` - удаляет из памяти shell переменную или функцию
- `wait` - ожидает окончания дочернего процесса
- `ls -li, stat` - утилиты, ко которым можно обратиться к таблице файловых дескрипторов (как [[stat, fstat|вот эти]])
- `mount` - утилита для получения файловой системой идентификатора
