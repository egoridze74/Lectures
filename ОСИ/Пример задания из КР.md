## Условие
``` c++
int main()
{
	int a = 1, b = 2, c = 3, d = 4, p[2], f[2];
	char buf[80];
	memset(buf, 0, 80);
	pipe(p);
	if (fork() == 0)
	{
		creat("a.txt", 0432);
		dup2(5, 1);
		printf("hello");
		a = write(1, "Boys ", 4);
		b = open("a.txt", 0);
		c = read(b, buf, 3);
		pipe(f);
		if (fork() == 0)
		{
			a = read(a, buf, 4);
			d = dup(1);
			b = read(6, buf, 3);
			exit(1);
		}
		wait(&c);
		exit(3);
	}
	else
	{
		wait(&a);
		b = open("a.txt", 0);
		c = read(b, buf, 10);
		printf("Many");
		d = read(3, buf, 10);
		exit(0);
	}
}
```
У каждого на КР будет приблизительно такой фрагмент кода. Надо будет определить, чему будут равны значения переменных и массивов после его выполнения при условии, что файл "a.txt" в текущем каталоге не существует.

## Пример анализа
1) Сначала выделили память для переменных и массивов
2) memset очищает память (это признак хорошего тона)
3) Будем параллельно вести таблицу дескрипторов

| таблица отца | дескриптор       |
| ------------ | ---------------- |
| 0            | $/dev/tty_{чт}$  |
| 1            | $/dev/tty_{зап}$ |
| 2            | $/dev/tty_{зап}$ |
`printf()` обращается к дескриптору с id = 1
`error()` обращается к дескриптору с id = 2
4) Выполняется pipe, который создает файл межпроцессороного канала и открывает 2 записи в таблице

| таблица отца | дескриптор       |
| ------------ | ---------------- |
| 0            | $/dev/tty_{чт}$  |
| 1            | $/dev/tty_{зап}$ |
| 2            | $/dev/tty_{зап}$ |
| 3            | $pipe_{чт}$      |
| 4            | $pipe_{зап}$     |
5) Кроме того, будем вести таблицу значений переменных после каждого из процессов. Всего нам надо будет написать $5 * 3 = 15$ значений (5 переменных, 3 процесса). Пока что она будет такой%

| переменная | отец | сын | внук |
| ---------- | ---- | --- | ---- |
| a          | 1    | 1   | -    |
| b          | 2    | 2   | -    |
| c          | 3    | 3   | -    |
| d          | 4    | 4   | -    |
| buf        | 0    | 0   | -    |
6) Создали процесс сын:

| таблица сына | дескриптор       |
| ------------ | ---------------- |
| 0            | $/dev/tty_{чт}$  |
| 1            | $/dev/tty_{зап}$ |
| 2            | $/dev/tty_{зап}$ |
| 3            | $pipe_{p \ чт}$  |
| 4            | $pipe_{p \ зап}$ |
7) Вызываем creat - создаем файл и открываем его на запись, а потом навешиваем наши права доступа. Получается, что сейчас мы можем в него писать, но если закроем, то опять открыть на запись не сможем. После этого вызываем dup2(5, 1) и получим такую картину:

| таблица сына | дескриптор       |
| ------------ | ---------------- |
| 0            | $/dev/tty_{чт}$  |
| 1            | $a.txt_{зап}$    |
| 2            | $/dev/tty_{зап}$ |
| 3            | $pipe_{p \ чт}$  |
| 4            | $pipe_{p \ зап}$ |
| 5            | $a.txt_{зап}$    |
8) printf("Hello") запишет строку в стандартный вывод **через буфер**, то есть строка там сразу не появится.
9) write запишет строку "Boys" (пробел не впишется, потому что запишем только 4 символа) в файл стандартный вывод (сейчас у нас это $a.txt_{зап}$). В итоге в файле у нас пока что будет "Boys"
10) Присвоили a = write(...) и получили, что a = 4 (сколько байт записали)

| переменная | отец | сын | внук |
| ---------- | ---- | --- | ---- |
