#методы_проги 
### Идея
Похож на [[Сортировка выбором|сортировку выбором]]. Просматриваем массив и каждое встреченное число ставим на правильное место в уже отсортированной части массива.

### Алгоритм
1. Поочередно берем элемент, начиная с 1, и сравниваем его с элементами в отсортированной части справа налево.
2. Сдвигаем элементы отсортированной части вправо, пока не найдем позицию, куда нужно вставить текущий элемент.
3. Вставляем элемент на найденную позицию.
4. Повторяем шаги 2-4 для всех элементов массива.

### Реализация
```
arr = [...]
n = len(arr)

for i in range(1, n):
    key = arr[i]
    j = i - 1
    
    while j >= 0 and arr[j] > key:
        arr[j + 1] = arr[j]
        j -= 1
    
    arr[j + 1] = key
```

### Анализ
1. Алгоритмическая сложность: $O(n^2)$
2. Дополнительные затраты по памяти: $O(1)$
3. Устойчивость: да
4. Естественность поведения: да, но не всегда
