### Идея
Тоже "разделяй и властвуй", но чисто по положению. Делим массив напополам

### Алгоритм
1. Если массив состоит из 0 или 1 элемента, он уже отсортирован.
2. Находим середину: `mid = n // 2`.
3. Рекурсивно сортируем левую часть (`arr[0:mid]`).
4. Рекурсивно сортируем правую часть (`arr[mid:n]`).
5. Сливаем две отсортированные половины в один отсортированный массив

### Реализация
```
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
        
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    
    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0
    
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
            
    result.extend(left[i:])
    result.extend(right[j:])
    return result
```

### Анализ
1. Алгоритмическая сложность: $O(n \cdot \log_2 n)$
2. Дополнительные затраты по памяти: $O(n)$
3. Устойчивость: да
4. Естественность поведения: нет
