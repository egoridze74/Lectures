### Идея
Юзаем [[Сортировка пузырьком|сортировку пузырьком]], каждый раз меняя направление прохода. Когда встретились в середине - заканчиваем.

### Алгоритм
1. Устанавливаем границы неотсортированной части: `left = 0`, `right = n - 1`.
2. **Проход слева направо (`left` → `right`)**: как в обычном пузырьке, толкаем максимальный элемент в `right`. После прохода `right` уменьшаем на 1.
3. **Проход справа налево (`right` → `left`)**: толкаем минимальный элемент в `left`. После прохода `left` увеличиваем на 1.
4. Повторяем шаги 2 и 3, пока `left < right`

### Реализация
``` python
arr = [...]
n = len(arr)
left = 0
right = n - 1

while left < right:
    for i in range(left, right):
        if arr[i] > arr[i + 1]:
            arr[i], arr[i + 1] = arr[i + 1], arr[i]
    right -= 1

    for i in range(right, left, -1):
        if arr[i - 1] > arr[i]:
            arr[i - 1], arr[i] = arr[i], arr[i - 1]
    left += 1
```

### Оцениваем по критериям:
1. Алгоритмическая сложность: $O(n^2)$
2. Дополнительные затраты по памяти: $O(1)$
3. Устойчивость: да
4. Естественность поведения: да
