### Идея
Толкаем самое большое число вправо

### Алгоритм
Идем слева направо и смотрим на пару числе. Если левое больше или равно правому, то меняем их местами. Когда дошли до тупика - начинаем заново с самого первого числа, но на более коротком промежутке.

Можно менять ориентацию (идти справа налево). Это и реализуем.

### Реализация
``` Python
arr = [...]
n = len(arr)

for i in range(n):
	for j in range(n - 1, ):
		if a[j - 1] > a[j]:
		x = a[j - 1]
		a[j - 1] = a[j]
		a[j] = x
```

### Оцениваем по критериям:
1. Алгоритмическая сложность: $O(n^2)$
2. Дополнительные затраты по памяти: $O(1)$
3. Устойчивость: да
4. Естественность поведения: нет
